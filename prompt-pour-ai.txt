## 🎯 OBJECTIF
Corriger et améliorer le site QA Universe — un site web statique monopage (single-file HTML/CSS/JS) dédié au monde du QA/Testing. Tu trouveras le code source en section RESSOURCES ci-dessous.

## 📋 CONTEXTE
- C'est un fichier HTML unique (~11 500 lignes, ~490 KB) avec tout le CSS inline dans <style> et tout le JS inline dans <script>
- Le site est hébergé sur GitHub Pages (recette : https://atmantest.github.io/recette-qa-universe/, prod : https://atmantest.github.io/qa-universe/)
- L'utilisateur est Thasin Jahangir, Consultant QA Senior, il parle français

- L'agent principal (Hermes Agent) appliquera tes corrections localement, lancera les tests, commitera et pushera

## 🔴 PROBLÈMES SIGNALÉS À CORRIGER (par priorité)

### 1. RESPONSIVE MOBILE — CORRECTIONS INCOMPLÈTES
Le burger menu ne fonctionnait pas (corrigé en partie) mais plusieurs bugs persistent :
- **Pages tronquées** : quand on clique sur un onglet (Dashboard, Tools, ISTQB Prep, etc.) sur mobile, le contenu dépasse / est coupé
- **Chat panel tronqué** : la fenêtre de l'assistant IA 🤖 (position:fixed, right:88px) est trop grande et coupée sur mobile
- **TJ avatar** : le cercle "TJ" dans la navbar doit être caché sur mobile (il prend de la place inutilement)
- **Navbar overflow** : les icônes dans .nav-actions débordent sur mobile

### 2. MODE LIGHT — CORRECTIONS
Le mode clair [data-theme="light"] existe en CSS mais :
- Certains éléments gardent leur fond sombre
- Les cartes, inputs, et textes ne sont pas correctement stylés en mode light
- L'assistant IA 🤖 n'a pas de styles light mode (fond blanc, texte foncé, bordures adaptées)

### 3. ASSISTANT IA 🤖 — AMÉLIORATIONS
- La vibration toutes les 10s doit fonctionner (actuellement l'animation CSS peut être écrasée par d'autres états)
- Les questions prédéfinies doivent défiler horizontalement sur mobile
- Le design doit ressembler à un vrai assistant (bulles de chat, etc.)
- Les 10 réponses prédéfinies sont bonnes mais peuvent être enrichies

### 4. GÉNÉRAL
- Vérifier que tous les éléments data-i18n s'affichent correctement
- Vérifier que la bascule dark/light fonctionne sur TOUS les composants (footer, assistant, drawer, tools cards, hero card)
